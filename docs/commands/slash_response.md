# Command: slash_response

Send a response to a slash command as a message in the specified channel.

This command is typically used in plugin handlers to send responses back as regular messages with interaction information.

## Request

```json
{
  "cmd": "slash_response",
  "channel": "<channel_name>",
  "response": "<response_text>",
  "command": "<command_name>",
  "invoker": "<user_id>"
}
```

### Fields

- `channel`: (required) Channel name where the response message should be posted.
- `response`: (required) The response text to send as a message.
- `command`: (required) The name of the slash command being responded to (without `/` prefix).
- `invoker`: (optional, default: current user's ID) User ID who invoked the original slash command.

## Response

### On Success

```json
{
  "cmd": "slash_response",
  "message": {
    "id": "message-uuid",
    "user": "username",
    "content": "Response text here",
    "timestamp": 1722510000.123,
    "type": "message",
    "pinned": false,
    "interaction": {
      "command": "command_name",
      "username": "invoker_username"
    }
  },
  "channel": "<channel_name>"
}
```

### Message Properties

The saved message has these properties:
- `id`: Auto-generated UUID
- `user`: The invoker's username (converted from user ID)
- `content`: The response text
- `timestamp`: Current Unix timestamp
- `type`: Always `"message"`
- `pinned`: Always `false`
- `interaction`: Object containing command interaction details
  - `command`: The name of the slash command that generated this response
  - `username`: The username of the user who invoked the command

**Important**: This response is **not global** - it's sent only to the requesting client. The message is saved to the channel history for all users to see later.

## Error Responses

- `{"cmd": "error", "val": "Authentication required"}`
- `{"cmd": "error", "val": "User roles not found"}`
- `{"cmd": "error", "val": "Channel parameter is required for slash commands"}`
- `{"cmd": "error", "val": "Command parameter is required for slash responses"}`
- `{"cmd": "error", "val": "Slash response must be a string"}` - `response` is not a string
- `{"cmd": "error", "val": "Slash response cannot be empty"}` - `response` is empty or whitespace

## Notes

- User must be authenticated.
- The response is saved as a regular message in the channel's message history.
- All users with channel access can view the message (not just the invoker).
- The message is attributed to the `invoker` (or current user if not provided).
- `response` text is trimmed of leading/trailing whitespace.
- **The slash_response is not broadcast globally** - only the requesting client receives the response.
- The `interaction` property allows clients to identify that this message was generated by a slash command.
- User IDs are converted to usernames before sending to clients.

## Interaction Property

The `interaction` property is key for clients to properly render slash command responses:

```json
{
  "interaction": {
    "command": "weather",
    "username": "alice"
  }
}
```

This tells clients:
- This message was generated by the `/weather` command
- The command was invoked by user "alice"

Clients can use this information to:
- Display special UI for command responses
- Show which command was executed
- Allow users to see who invoked the command
- Apply different styling or behavior for command responses vs regular messages

## Non-Global Behavior

Unlike previous versions, slash_response is **not global**:
- Only the client that sent the slash_response request receives the response
- Other clients in the channel will see the new message when they fetch channel messages via `messages_get`
- The message is saved to persistent storage
- This allows for more efficient communication, avoiding unnecessary broadcasts

## Message Lifecycle

```
[Plugin receives slash_call event]
        ↓
[Plugin executes command logic]
        ↓
[Plugin sends slash_response]
        ↓
[Server validates request]
        ↓
[Server creates message with interaction]
        ↓
[Server saves message to channel]
        ↓
[Server sends response to plugin]
        ↓
[Other users see message when fetching messages]
```

## Plugin Handler Example

### Basic Response

```python
def handle_slash_command(plugin, ws, data, server_data):
    """Handle slash command and send response"""
    command = data['val']['command']
    args = data['val']['args']
    invoker_id = data['invoker']
    channel = data['channel']
    
    # Execute command
    result = process_command(command, args)
    
    # Send response as message in channel with interaction info
    return {
        "cmd": "slash_response",
        "channel": channel,
        "response": result,
        "invoker": invoker_id,
        "command": command  # Required for interaction property
    }
```

### Response with Error Handling

```python
def handle_slash_command(plugin, ws, data, server_data):
    """Handle slash command with error handling"""
    try:
        command = data['val']['command']
        args = data['val']['args']
        invoker_id = data['invoker']
        channel = data['channel']
        
        # Validate command
        if command not in available_commands:
            return {
                "cmd": "slash_response",
                "channel": channel,
                "response": f"Unknown command: /{command}",
                "invoker": invoker_id,
                "command": command
            }
        
        # Execute command
        result = execute_command(command, args)
        
        # Send success response
        return {
            "cmd": "slash_response",
            "channel": channel,
            "response": str(result),
            "invoker": invoker_id,
            "command": command
        }
        
    except Exception as e:
        Logger.error(f"Command error: {e}")
        return {
            "cmd": "slash_response",
            "channel": channel,
            "response": f"Error executing command: {str(e)}",
            "invoker": invoker_id,
            "command": command
        }
```

### Rich Response

```python
def handle_slash_command(plugin, ws, data, server_data):
    """Handle slash command with rich response"""
    command = data['val']['command']
    args = data['val']['args']
    invoker_id = data['invoker']
    channel = data['channel']
    
    # Process command
    if command == "weather":
        city = args.get("city", "London")
        weather_data = get_weather(city)
        
        # Format rich response
        response = f"Weather in {city}:\n"
        response += f"Temperature: {weather_data['temp']}°C\n"
        response += f"Condition: {weather_data['condition']}\n"
        response += f"Humidity: {weather_data['humidity']}%"
        
    elif command == "user_info":
        target = args.get("user", invoker_id)
        user = get_user_info(target)
        response = f"User: {user['username']}\n"
        response += f"Roles: {', '.join(user['roles'])}\n"
        response += f"Joined: {user['join_date']}"
    
    else:
        response = "I can help with `/weather` or `/user_info`"
    
    return {
        "cmd": "slash_response",
        "channel": channel,
        "response": response,
        "invoker": invoker_id,
        "command": command
    }
```

## Client-Side Handling

### Displaying Messages with Interaction

```javascript
function renderMessage(message) {
  if (message.interaction) {
    // This is a slash command response
    return `
      <div class="message slash-response">
        <div class="header">
          <span class="command-badge">/${message.interaction.command}</span>
          <span class="invoker">@${message.interaction.username}</span>
        </div>
        <div class="content">${escapeHtml(message.content)}</div>
      </div>
    `;
  } else {
    // Regular message
    return `
      <div class="message regular">
        <span class="username">${message.user}</span>
        <span class="content">${escapeHtml(message.content)}</span>
      </div>
    `;
  }
}
```

### Handling Slash Response

```javascript
ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  if (data.cmd === 'slash_response') {
    // The plugin sent a response
    // Add the message to the channel
    addMessageToChannel(data.channel, data.message);
    
    // Show it in the UI immediately
    renderMessage(data.message);
  }
};
```

    "timestamp": 1234567890.123
  },
  "global": true
}
```

    "timestamp": 1234567890.123,
    "interaction": {
      "command": "weather",

    }

```

## Best Practices

1. **Always include the command parameter**: Without it, clients won't know which command generated the response
2. **Use the invoker parameter**: Ensures the message is attributed to the correct user
3. **Handle errors gracefully**: Return helpful error messages in the response
4. **Keep responses concise**: Long responses may be difficult to display nicely
5. **Use proper formatting**: Include newlines or structure for complex responses
6. **Sanitize input**: Always validate and sanitize command arguments
